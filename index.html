Naming Conventions & Commit Messages

Naming Conventions

Use all lowercase: Different operating systems treat capitalization differently.
Good exmaple: ten-tree-logo.svg
Bad example: Today-in-History.html
No spaces: URLs cannot have spaces in them
Good example: chocolate-cake-recipe.html
Bad example: chocolate cake recipe.html
Only letters, numbers, and dashes: URLs use limited amounts of charcaters. Dashes ae word seperators, underscores are word connectors.
Good example: west-coast-art-2021.html
Bad example: #1-blueberry_muffin-recipe.html

How to Commit Messages to Github

  • Minimum 3 words and 10 characters
  • Proper spelling and grammar
  • Capital starting letter, no period
  • Must start with an imperative verb (ex. Create, fix, add, solve, etc.)
    • Pretend every commit starts with the phrase: "This commit will..."

Commonly Used HTML Tags

  • (h1) to (h6) tags: Used to mark headings according to their importance. H1 being the most important and biggest, while H6 is the least important and smallest.
  • (p) tags: Defines a paragraph of text.
  • (ul) and (li) tags: This is an unordered list tag and a list item tag. This creates a bullet list
  • (ol) and (li) tags: This is an ordered list tag and a list item tag. This creates a numbered list.
  • (dl), (dt) and (dd) tags: This is a description list, a term or name, and a term description.
  • (q) tags: used to mark up quotes embedded in other elements, like a paragraph.
  • (blockquote) tags: For large stand-alone quotes.
  • (cite) tags: Marking the source of the quote.
  • (em) tags: Used to emphasize a word or sentence.
  • (strong) tags: Used for lots of emphasis, has strong importance and urgency.
  • (i) tags: Italicizes a wor or sentence. Used for another language, technical term, or title.
  • (b) tags: Makes a keyword bold.
  • (a href="") link text (/a) tag and attribute: The a tells where the link should start and end. To create the hyperlink, the href attribute must be used.
  • (img src="" alt=""/) attributes: This is used to link an image to the website by the HTML document. These two attributes specify the extra detail about the image, meaning the location of the image and the alternative content.

HTML Structuring Content

  • (header): It contains the introductory content which is usually found at the top of the page. It would typically include a logo or company name and main navigation.
  • (footer): Found at the bottom of the page. This often includes lists of links, copyright information, extra information about the company, terms and services.
  • (main): This wraps around the main content of the page. It is only used once per page and tells the browser that this is where to find the main content.
  • (section): This is used to wrap around related pieces of content. Usually, each section has its own heading.
  • (article): This represents a self-contained composition in a web page. Which is intended to be independently distributable or reusable.
  • (nav): This is the section of a page whose purpose is to provide navigation links. Either within the current document or to other documents.
  • (aside): This represents a portion of a document whose content is only indirectly related to the document’s main content. This content will frequently be presented as a sidebar on a website.
  • (a href="#...") and (section id="..."): This is how to link to an element on the same page.
  • (!--write your comment here--!): These are comments, they are not displayed by the browser, but they can help document your HTML source codes.

Landmark Roles and Accessibility

  • (header role="banner"): This is for defining an element as the masthead, which is added to the primary (header).
  • (nav role="navigation"): This is for defining an element as the primary navigation, which is added to the top of (nav).
  • (main role="main"): This is for defining an element as being the main content of the page, which is added to the (main).
  • (aside role="complimentary"): This is for defining an element as being content related to the primary content, which is usually added to an (aside).
  • (footer role="contentinfo"): This is for defining an element as information about the content. For example, copyright, terms, privacy policy. Which is usually added to a (footer).
  • (a href="#" aria-label="..."): This is an attribute that provides the user with a non-visual label that will only be announced by screen readers. It adds a description of the link.
  • (img src="" alt="" role=presentation)): Always describes an image with the alt attribute. If purely decorative you can leave it empty and apply role=“presentation”.

Paths and URLs

  • Paths in the same folder: From index.html to plesiosaus.html is plesioaurs.html
  • Moving into folders: From index.html to logo.svg is image/logo/svg
  • Moving out of folders: From apatosaurus.html to logo.svg is ../images/logo.svg
  • /:This is used when moving into folders.
  • ../: This is used when moving out of folders. Add another (.../) each time you move out of a folder.

How to Setup CSS

  • CSS file: Attach within the (head) of the html document using the (link) element.
  • rel: Used to define this item as a stylesheet
  • href: Used to point to the location of the CSS file.
  • Rule set: Composed of a selector and one or many declration surrounded by curly braces.
  • Selector: Identifies which html element should be targeted for styling.
  • Declaration: Assigns a value to a property, seperated by a colon. Each declaration must end with a semi-colon.
  • Property: The type of design you want to add. (ex. width, color, border, etc.)
  • Value: Accepted value for the property. (ex. 30px, blue, etc.)
  • /*______*/: This is used to comment in CSS. (To write this you can also hold command and press /.

Universal Selectors

Element Selector: Select all the tags of the same type.
html {
backround-color: purple;
}
Targeting Multiple Elements: If you’d like to style more than one element the same way you can use a comma to separate the tag names.
h1, h2, h3 {
background-color: blue;
}
Class Selector: Select and element that has a class.
.card-text {
padding: 0 1em 1em;
}
ID Selector: Select an element that has the ID - remember you can only have one unique ID per page.
#section {
background-color: mistyrose;
}
Descendant Selector: Select an element that's a descendant of another element. Refers to elements nested within other elements. To create a descendant selector we'll need to use two or more selectors separated by white space.
nav li {
color: #fff;
}
Child Selectors: Select and element directly inside another element.
div > h3 {
text-transform: uppercase;
}
Adjacent Sibling: Select an element immediately beside another element.
h2 + p {
color: orchid;
}
General Sibling: Select an element that's at the same level.
h3 ~ a {
padding: .25em .75em;
}
Combine Multiple Type of Selectors:
#section > h2 ~ p strong {
color: green;
}
Attribute: Select an element by it's attribute.
A[target="blank"] {
font-weight: bold;
color: #f80;
}

Pseudo Classes

first-child: Select the element when it's the first inside it's parent.
ul li: first-child {
background-color: #ff4858;
}
last-child: Select the element when it's the last inside it's parent.
ul li: last-child {
background-color: #00ccc0;
}
nth-child(): Select and element by it's number.
li: nth-child(3) {
background-color: purple;
}
first-of-type: Select an element that's the first of it's kind within its parent.
p: first-of-type {
font-style: italic;
}

Styling Text

Font Properties

font-size: For adjusting the size of the font. A single number with a unit (rem [best choice], em, px).
ex. 2rem, 1.25em, 24px
font-weight: Sets how thick or thin characters in text should be displayed.
ex. bold, normal, 700
font-style: Specifies font style for text.
ex. italic or normal
line-height: For adjusting the line-height. A single number, a miltiplier that multiplies the font-size.
ex. 1.5

Text Decorations

text-transform: Change capitalization such as upper case, lowercase, capitalize. It’s a good idea to use CSS to make all caps instead of writing it directly in HTML for accessibility purposes.
ex. uppercase, lowercase, capitalize
text-decorations: Specifies the decoration added to text. The only things in a website that should be underlined are links.
ex. none, underline, line-through, overline
text-align: Specifies the horizontal alignment of text in an element.
ex. left, center (American spelling), right, justify
text-indent: Indents the first line of text like traditionally printed materials.
ex. 1rem, 2em, 20px, 25%
text-shadow: Adding a shadow to the text. Specified like this:
text-shadow: horizontal-offset vertical-offset blur-radius color
ex. text-shadow: 2px 2px #ff0000;

Styling Links

Unvisited Link: For styling the default link state.
a:link {
color: orange;
}
Visited Link: For styling the link after it's been visited by the user.
a:visited {
color: limegreen;
}
Mouse Over Link: For styling the link when the mouse is sitting over it.
a:hover {
color: purple;
}
Focus Link: For styling the link when the keyboard has tab-focused it.
a:focus {
color: blue;
}
Selected Link: For styling the link when the mouse button is pressed down on it.
a:active {
color: yellow;
}
Add same style to more than one link state.
a:link,
a:visited {
color: yellow;
}

Everything is a Box

The CSS Box Model

The box model is a box that wraps around every HTML element; made up of four parts: content, padding, border and margins.

  • Content: First layer, defines the basic box.
  • Padding: Outside content, pushing the edge of the box away from the content.
  • Border: Stroke at the edge of padding and aroun th edge of the box. Background colour ends here.
  • Margin: It is outside of the box, the layer pushing other boxes away. Space around an element that seperates it from other elements.
  • Box Model Tool With Dev Tools: If you inspect and element on a browser, you can see the size of the element and its margin, padding, and border.

Padding and Margins

These two properties are used to apply space, either inside te box or outside the box.

Padding: Adds spacing inside the box, pushing content away from the border.
.box {
padding-top: 0;
padding-right: 30px;
padding-bottom: 40px;
padding-left: 4em;
}
Margin: Invisible space around the box. Adds spacing outside the box, pushing away other boxes away from the box.
.box {
margin-top: -10px;
margin-right: 30px;
margin-bottom: 40px;
margin-left: 4em;
}
Shorthands: Start at the top and make your way clockwise around the box. If you leave numbers from the list, the browser will match the opposing sides value for any mossing numbers.
.box {
padding: 0 30px 40px 4em;
margin: -10px 30px 40px 4em;
}
Margin Collapsing: The bottom value of one box can collapse into the top value of the next. The browser choosing only the highest of the two.
The distance between these is not 40px. Instead, the browser picks the bigger f the two and chooses it. The margin between these boxes is 25px

Width and Height

This is used to set the height and width of an element.

%: Use percent for elements that should grow with the size of the browser.
ex. columns, images
em: Use ems for elements that should grow with the size of the text.
ex. line-length
px: Use pixels for things that should never change size.
ex. logos, icons
min-width: Force the box to be at least this wide, but can grow wider.
min-height: Force the box to be at least this tall, but can grow taller.
max-width: Force the box to never exceed this width, but can get narrower than it.
max-height: Force the box to never exceed this height, but can get shorter than it.

Border Box

  • Default Box Model: The padding and border add onto the width of the box, making it wider.
  • Border Box: The padding and border are inside the width of the box, so the total width is the same as the value width.
  • Always put this at the top of every CSS file.

Display Properties

This sets the display of an element. It manages and alters how the HTML elements are shown on the web page.

display: inline;
Allows elements on the same line; height and width properties have no effect.
Allows other elements to fit beside them on the same line.
Takes up only as much width as needed.
Width, margin, padding don't really work.
display: block;
Force the element to be on its own line regardless of its width.
Takes up its own line, anything that comes after goes onto the next line.
Takes up the whole width of its parent element by default.
Width, margin, padding works as expected.
display: inline-block;
Element is formatted as an iline element but you can apply height and widt values.
Fits beside other elements on the same line (from inline).
Takes up only as much space as needed (from inline).
Width, margin, padding work as expected (from block).
display: none;
The element is completely removed.
This completely hides the element from the screen and any other accessibility tools.

Centering Elements

text-align: center; On inline and inline-block elements.
If applied to the element itself, it will only center the content inside of the box.
If applied to the parent, it will center the boxes within the page.
text-align: center; On block elements.
This only affects the content inside it, but will not move the element box itself to the center of its parent.
margin: 0 auto; On block elements.
If there is a block element with a width, and want it to center within its parent, use auto margins.
Auto: This tells the browser to automatically calculate a margin that's the same on the left and right.

Floats and Clear

This allows text to wrap around another HTML element.

  • float: left; Moves the element to the left, allowing text to wrap on the right side.
  • float: right; Moves the element to the right, allowing text to wrap on the left side.
  • float: none; Would turn floating off.
  • Wrapping with float: To push the text away from floated element, margin will work well.
  • clear: left; Goes below all left floated elements.
  • clear: right; Goes below all right floated elements.
  • clear: both; Goes below all floated elements.

Flex Box

Controls the flow and alignment of elements for more complex layouts.

Works on Children: Many of the flexbox properties are applied to the parent element, but affect the child elements.
The Flex Container: To create a flex container, we select the parent element in our CSS and set the value to display: flex; Now all direct children are considered flex items.
section {
display: flex;
}
When Flex is Applied: Some initial values will be defined. Flex items will behave in the following ways:
Items display in a row (flex-direction property's default is row).
The items start from the start edge of the main axis.
The items will stretch to fill the size of the cross axis.
The flex-wrap property is set to nowrap.
flex vs. inline-flex: Setting our flex containers to behave like block or inline-block elements.
Flex Direction: Change the orientation in how your conten is displayed, justified and aligned.
This establishes the main-axis, which defines the direction flex items are placed in the flex container.
Flexbox is a single-direction layout concept.
Wrapping Flex Items: With this property, you can control wether the flex container is a single -line or multi-line layout.
The flex container can break flex items onto multiple lines and allow them to wrap as needed.
justify-content: Lets you control the position and alignment of flex container items on the main axis, and how much space is between flex items
The default value is flex-start, which places items toward the start of each flex line.
Other accepted values include: flex-end, center, space-between, space-around and space-evenly.
flex-start: The default position which will line items up at the start edge of the container.
flex-end: Line items up at the end of the container.
center: Move all of the items into the center of the flex container.
space-between: Items touch the start & end edges of the flex container then envenly distributes the space between.
space-around: Distributes items evenly with equal space around each item (doubled up between each item).
space-evenly: Distributes items with equal space around all items.
align-items: This works on the cross axis.
Accepted values include: stretch, flex-start, flex-end and center.
Note: With align-items applied to the flex container, all direct children will use the alignment. If you set one or more of the child items using align-self, it will override property set on the flex container for those specific instances.
stretch: The initial value for the property and why flex items stretch to the height of the container by default.
flex-start: Line items up at the start of a container.
flex-end: Line items up at the end of a container.
center: Line items in the center.
align-content: Aligns a flex containers lines when there is extra space on the cross-axis.
Accepted values include: normal, flex-start, flex-end, center, space-between, space-around, space-evenly and stretch.
Note: align-content requires extra space (height) to exist in order to be applied and this property has no effect on single line flex containers (i.e. ones with flex-wrap: nowrap).
normal: The default with the items packed in their default position as if no value was set.
flex-start: Items are packed to the start of the container.
flex-end: The items are packed to the end of the container.
center: The items are packed centered int the container.
space-between: Items are evenly distributed; first line is at the start of the container, last one at the end.
space-around: Items are evenly distributed with equal space around each line.
space-evenly: Items are evenly distributed wih equal space around them.
stretch: Lines are stretched to take up the remaining space.
