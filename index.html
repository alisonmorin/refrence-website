Naming Conventions & Commit Messages

Naming Conventions

Use all lowercase: Different operating systems treat capitalization differently.
Good exmaple: ten-tree-logo.svg
Bad example: Today-in-History.html
No spaces: URLs cannot have spaces in them
Good example: chocolate-cake-recipe.html
Bad example: chocolate cake recipe.html
Only letters, numbers, and dashes: URLs use limited amounts of charcaters. Dashes ae word seperators, underscores are word connectors.
Good example: west-coast-art-2021.html
Bad example: #1-blueberry_muffin-recipe.html

How to Commit Messages to Github

  • Minimum 3 words and 10 characters
  • Proper spelling and grammar
  • Capital starting letter, no period
  • Must start with an imperative verb (ex. Create, fix, add, solve, etc.)
    • Pretend every commit starts with the phrase: "This commit will..."

Commonly Used HTML Tags

  • (h1) to (h6) tags: Used to mark headings according to their importance. H1 being the most important and biggest, while H6 is the least important and smallest.
  • (p) tags: Defines a paragraph of text.
  • (ul) and (li) tags: This is an unordered list tag and a list item tag. This creates a bullet list
  • (ol) and (li) tags: This is an ordered list tag and a list item tag. This creates a numbered list.
  • (dl), (dt) and (dd) tags: This is a description list, a term or name, and a term description.
  • (q) tags: used to mark up quotes embedded in other elements, like a paragraph.
  • (blockquote) tags: For large stand-alone quotes.
  • (cite) tags: Marking the source of the quote.
  • (em) tags: Used to emphasize a word or sentence.
  • (strong) tags: Used for lots of emphasis, has strong importance and urgency.
  • (i) tags: Italicizes a wor or sentence. Used for another language, technical term, or title.
  • (b) tags: Makes a keyword bold.
  • (a href="") link text (/a) tag and attribute: The a tells where the link should start and end. To create the hyperlink, the href attribute must be used.
  • (img src="" alt=""/) attributes: This is used to link an image to the website by the HTML document. These two attributes specify the extra detail about the image, meaning the location of the image and the alternative content.

HTML STructuring Content

  • (header): It contains the introductory content which is usually found at the top of the page. It would typically include a logo or company name and main navigation.
  • (footer): Found at the bottom of the page. This often includes lists of links, copyright information, extra information about the company, terms and services.
  • (main): This wraps around the main content of the page. It is only used once per page and tells the browser that this is where to find the main content.
  • (section): This is used to wrap around related pieces of content. Usually, each section has its own heading.
  • (article): This represents a self-contained composition in a web page. Which is intended to be independently distributable or reusable.
  • (nav): This is the section of a page whose purpose is to provide navigation links. Either within the current document or to other documents.
  • (aside): This represents a portion of a document whose content is only indirectly related to the document’s main content. This content will frequently be presented as a sidebar on a website.
  • (a href="#...") and (section id="...")This is how to link to an element on the same page.
  • (!--write your comment here--!):These are comments, they are not displayed by the browser, but they can help document your HTML source codes.

Landmark Roles and Accessibility

  • (header role="banner"): This is for defining an element as the masthead, which is added to the primary (header).
  • (nav role="navigation"): This is for defining an element as the primary navigation, which is added to the top of (nav).
  • (main role="main"): This is for defining an element as being the main content of the page, which is added to the (main).
  • (aside role="complimentary"): This is for defining an element as being content related to the primary content, which is usually added to an (aside).
  • (footer role="contentinfo"): This is for defining an element as information about the content. For example, copyright, terms, privacy policy. Which is usually added to a (footer).
  • (a href="#" aria-label="..."): This is an attribute that provides the user with a non-visual label that will only be announced by screen readers. It adds a description of the link.
  • (img src="" alt="" role=presentation)): Always describes an image with the alt attribute. If purely decorative you can leave it empty and apply role=“presentation”.

Paths and URLs

  • Paths in the same folder: From index.html to plesiosaus.html is plesioaurs.html
  • Moving into folders: From index.html to logo.svg is image/logo/svg
  • Moving out of folders: From apatosaurus.html to logo.svg is ../images/logo.svg
  • /:This is used when moving into folders.
  • ../: This is used when moving out of folders. Add another (.../) each time you move out of a folder.

How to Setup CSS

  • CSS file: Attach within the (head) of the html document using the (link) element.
  • rel: Used to define this item as a stylesheet
  • href: Used to point to the location of the CSS file.
  • Rule set: Composed of a selector and one or many declration surrounded by curly braces.
  • Selector: Identifies which html element should be targeted for styling.
  • Declaration: Assigns a value to a property, seperated by a colon. Each declaration must end with a semi-colon.
  • Property: The type of design you want to add. (ex. width, color, border, etc.)
  • Value: Accepted value for the property. (ex. 30px, blue, etc.)
  • /*______*/: This is used to comment in CSS. (To write this you can also hold command and press /.
